Scripts for Fargo and Carbonfin Outliner Interoperability

Tweet This Post

Carbonfin Outliner is an outliner for iOS that syncs with Dropbox and exports in OPML. Unlike Fargo, this app works off-line. Fargo works well for todo lists (thanks to the use of FontAwesome icons like and ), as does Outliner, which natively uses checkboxes. Unfortunately, Outliner does not preserve the import of attributes that it does not support. (The attributes it supports: _status and _note.) However, Fargo will preserve attributes created in Outliner, and with the help of some scripts, one can move the same OPML file between these two apps without losing any key information.

Moving Your OPML File Between Apps

It's easy to move an OPML file between these two apps. Just go to Dropbox.com, and find either your /Outliner folder or your /Apps/Fargo folder. Right-click on the file, and tell Dropbox to move it to the folder of the destination app. Or you can just drag the file from one folder to the other on your hard drive.

If you're moving from Fargo to Outliner, you will need to sync Outliner with Dropbox for the file to show up.

If you're moving from Outliner to Fargo, you will need to use the File > Open menu to locate the new file.

Four Scripts and Some CSS

For complete interoperability, four scripts are required.

  • Convert Outliner to Fargo

    • After importing an OPML file into Fargo, checkboxes are added to all headlines except for any line with the text "Archive." Status is maintained, so that if a box was checked in Outliner, it's checked in Fargo. If you wrote a note in Outliner, a small icon () shows up at the right of the headline. You must run this script every time you import a file from Outliner to Fargo.

  • todo

    • When you are adding new todo items in Fargo, the todo script works as follows:

    • Write your task.

    • Choose "todo" from menu.

    • An empty checkbox is added to the headline.

    • When you've completed the task, choose "todo" again. The box is now checked, and two other attributes are added to this headline: completed (with a date stamp) and _status ("checked"). The latter attribute is for Outliner to know that the task was completed. The timestamp will not carry over into Outliner, so be prepared to lose this information if you move the file between apps.

    • If you don't want to see the task anymore, choose "todo" again, and it is moved to the Archive, which has to be the final first-order headline in your outline.

  • show note

    • If you want to view a note in Fargo (you'll know there's a note because you'll see the note icon), use the "show note" script.

  • add note

    • If you want to add a note in Fargo such that it shows up as a note in Outliner, use the "add note" script.

  • Finally, for the note icon to be visible when there is a note, add some CSS in the Settings. Click on your name in the top right of Fargo, click on Settings, then drop the CSS in the CSS field.

  • Put the first four headlines in your menubar outline. Put the final headline in your CSS.

  • Visit the Fargo scripting page in the docs to learn how to install this script, or watch this video.
