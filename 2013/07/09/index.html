My Tabbed River

I got it to work!

07/09/13; 05:12:34 PM
An Idea for River4

It seems weird that I'm going through the trouble of running the OPML Editor 24/7 in an Amazon EC2 instance just so that I can publish a json file to an Amazon S3 bucket. The river html file can be published anywhere as long as the path to the json file is correct.

It would be cool if Trex -- the CMS part of Fargo -- could "read" an OPML file of RSS feeds within Fargo and automatically publish a json file every 15 minutes. Instead of worrying about the html, there could be a "river" template with an attribute pointing to the json file.

  • For example, I could have an OPML file of Fargo blog feeds, and name it "fargoblogs." In the Settings, there could be a checkbox like "convert this to json." Somewhere there's an output containing the URL to the json file.

  • I could then have a headline in a named outline called "Fargo River." In Edit Attributes, I create an attribute called "river" whose value is "http://static.scripting.com/river3/rivers/fargo.js"

  • Then I click on the icon, and boom! I see my river of news.

Basically, why does one have to have a server running all the time, whether on one's own computer or in an EC2 instance, and regardless of whether you're publishing the json and html files to Dropbox or an S3 bucket? Why can't it all be in Fargo?

07/09/13; 02:04:43 PM
I Just Don't Care Anymore ... About What I Used to Care About

I read my feeds using Fever, and am still very happy with it. However, I've noticed that one group of feeds I created -- I've called it "Minimalist Tech" -- just doesn't do it for me anymore. I call this group Minimalist Tech because it includes tech blogs that have minimalist blog themes, unlike, say The Verge or TechCrunch. Sites like Marco.org or Daring Fireball. Not too long ago, I was obsessed with iOS hacks, so much so that I blogged about them and religiously read sites like MacStories. It's not that I don't still get hacky -- I recently wrote a Drafts for iOS Dropbox action to add a YouTube embed to a Fargo blog -- but I find myself just skimming through these feeds and barely opening any of them.

Perhaps I've just found a workflow that meets my needs well enough that I don't have to keep looking.
