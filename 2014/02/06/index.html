How to Mark a Paperless Item as Complete Without Touching Your iOS Device#

Tweet This Post#

Plain text rocks, but I've been crushing on the list-making iOS app Paperless recently, and because it stores its data in XML files (at least when using Dropbox for sync), it's difficult to mark an item complete in a text editor. However, it's not impossible.#

I would not advise the following steps to anyone other than those who are comfortable editing XML files.#

To find the XML files, just go to the \Paperless folder in Dropbox. A typical list item will look like this:#

``#

` `#

` Cell phone bill`#

` $100`#

` NO`#

` NO`#

``#

To mark as complete, we have to do two things: #

1. Enter a value for ``#

2. Change the value for ``#

``#

` 2014-02-06 15:32:11 -0500`#

` Cell phone bill`#

` $100`#

` NO`#

` YES`#

``#

Above, I've changed the value of `` to YES. I've also entered a date string for ``. If you use TextExpander, use this data math for the snippet:#

%Y-%m-%d %H:%M:%S -0500#

The app uses +0000 UTC time, but above I've chosen -0500 because I live in New York. When I re-sync the app with Dropbox, it shows at the bottom: "Completed: Feb 6, 2014, 3:32 PM" so I know I did the date math correctly.#

Using the Launch Center Pro URL Parameter to Launch an Action from Another App#

Tweet This Post#

After reading [this Twitter conversation](https://twitter.com/welkerpatrick/status/430436207680835585) between [@welkpatrick](https://twitter.com/welkerpatrick), [@epramono](https://twitter.com/epramono) and [@Pgruneich](https://twitter.com/Pgruneich), I now have a deeper understanding of how `launch://?url=` works:#

@welkerpatrick launch://?url= will treat the URL as though you define it as a @LaunchCenterPro action. It will parse [list|a|b] correctly.

— Eric Pramono (@epramono) February 4, 2014
#

#

Combined with Federico Viticci's [Safari Action Menu In Launch Center Pro](http://www.macstories.net/tutorials/safari-action-menu-in-launch-center-pro/), it's all starting to come together: I can write a Launch Center Pro action, and use it from within another app entirely, _without having to have that action entered in LCP itself_.#

For example, I have an action in LCP to add an item and optional note to Paperless: #

paperless://addItem?itemName=[prompt:Item]&itemNote=[prompt:Note]&toListNamed=[list:Choose a list|List1|List2|List3]#

(You can view that URL Scheme documentation [here](http://blog.crushapps.com/2013/03/paperless-2-1-released/).)#

Now, if I've noted something in Drafts and later decide I want to send it to Paperless, I can just tap on the _paperlessList_ action in the Drafts Actions menu to launch LCP with a menu of lists already open. All I have to do is tap on the list to which I want to send the task, and the task and note are imported into Paperless.#

If you're in Drafts, tap on [this link](drafts://x-callback-url/import_action?type=URL&name=paperlessList&url=launch%3A%2F%2F%3Furl%3D%7B%7Bpaperless%3A%2F%2FaddItem%3FitemName%3D%5B%5Btitle%5D%5D%26itemNote%3D%5B%5Bbody%5D%5D%26toListNamed%3D%5Blist%3AChoose%20a%20list%7CList1%7CList2%7CList3%5D%7D%7D) to import the URL Action. And here is the action itself:#

launch://?url={{paperless://addItem?itemName=[[title]]&itemNote=[[body]]&toListNamed=[list:Choose a list|List1|List2|List3]}}#

The first line is the task, and any additional lines (if they exist) are added in the note field.#
